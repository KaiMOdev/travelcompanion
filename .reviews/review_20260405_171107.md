**[BUGS & CORRECTNESS] severity: medium**  
`voxlingo/server/index.ts:~480` — Culture API endpoint: IDs are overwritten, not preserved from LLM  
> The endpoint always overwrites the `id` field with a generated one (`${code}-${category}-${i + 1}`), even though the LLM is instructed to provide unique IDs. This could break client-side caching or referencing if the client expects stable IDs across requests or matches the LLM's IDs.  
> **Suggestion:** Only generate a fallback ID if the LLM's `id` is missing or not unique, or document that IDs are always server-generated.

**[BUGS & CORRECTNESS] severity: medium**  
`voxlingo/server/index.ts:~480` — Culture API endpoint: No validation for required fields (e.g., `speakable`, `romanized`)  
> The endpoint only checks for `id`, `title`, and `body`, but some categories require `speakable` and/or `romanized`. If the LLM omits these, the client may break or display incomplete data.  
> **Suggestion:** Add stricter validation per category, ensuring required fields are present and of the correct type.
