Data Flow — Voice Translation (Travel Mode)

Request Lifecycle
1️⃣ User taps mic → expo-av records 16-bit PCM, 16kHz mono
↓ WebSocket (socket.io)
2️⃣ Backend receives audio chunks → forwards to Gemini Live API
↓ WebSocket
3️⃣ Gemini translates speech → returns translated audio + text
↓ WebSocket
4️⃣ Backend relays response → caches translation in Firestore
↓ WebSocket
5️⃣ App plays translated audio + shows chat bubble with text
