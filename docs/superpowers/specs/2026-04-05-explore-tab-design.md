# Explore Tab — AI Local Guide

## Problem

VoxLingo helps travelers translate and understand culture, but has no way to discover **where to go**. Google Maps Places shows the same tourist-heavy results everywhere. Travelers want authentic, local-perspective recommendations — the spots locals actually go to, not the ones with the most reviews.

## Solution

A new **Explore tab** (4th tab, compass icon) that uses Gemini to generate vibe-based place recommendations for the selected destination. Each place comes with cultural context, contextual phrases, and deep links to VoxLingo's existing translation features.

## Key Differentiators

- **Vibe-based discovery** — search by mood ("Street Food", "Hidden Gems", "After Dark") not rigid categories
- **Cultural context built in** — each place includes etiquette tips, what to say, why it's special
- **Connected to translation** — tap a place to get context-specific phrases or start a live translation session
- **Free** — powered by existing Gemini backend, no Google Places API costs
- **Cached** — same 7-day server + client caching as culture content

## Screen Layout

### Top Section
- Destination banner with country name + flag (reuse existing pattern from Culture tab)

### Category Chips (horizontal scroll)
Reuse existing `CategoryChips` component pattern. Categories:

| Emoji | Category | Description |
|-------|----------|-------------|
| 🍜 | Street Food | Local food stalls, night markets, authentic eats |
| 🏛️ | Hidden History | Lesser-known temples, ruins, historical spots |
| ☕ | Chill Spots | Quiet cafes, parks, rooftop views |
| 🌙 | After Dark | Night markets, bars, evening activities |

### Place Cards (vertical scroll, paginated)
Each card displays:

- **Place name** — in English + local language (e.g., "Fuunji / 風雲児")
- **One-line description** — "Tiny tsukemen shop in an alley behind Shinjuku station"
- **Why it's special** — "Locals line up at 6am — the broth has been simmering since 1962"
- **Vibe tags** — small chips: casual, budget-friendly, romantic, family, etc.
- **Action buttons row**:
  - **Phrases** — opens PlacePhrases modal with 5-8 context-specific phrases
  - **Directions** — deep links to Google Maps / Apple Maps
  - **Translate** — opens Travel tab with destination language pre-selected

## Place Phrases Modal

When user taps "Phrases" on a place card:

- Modal overlay (similar to existing EmergencyCard pattern)
- Title: "Useful Phrases for [Place Name]"
- 5-8 contextual phrases generated by Gemini, specific to the place type
- Each phrase shows: English | Local language | tap-to-speak button (expo-speech)
- Examples for a ramen shop:
  - "I'd like the recommended dish" / おすすめをください
  - "No pork please" / 豚肉なしでお願いします
  - "Can I have more broth?" / スープを追加できますか
  - "The bill please" / お会計お願いします

## Data Model

### ExplorePlace type

```typescript
interface ExplorePlace {
  name: string;           // English name
  localName: string;      // Name in local language
  description: string;    // One-line description
  whySpecial: string;     // What makes it unique
  vibeTags: string[];     // e.g., ["casual", "budget-friendly"]
  phrases: PlacePhrase[]; // Context-specific phrases
  area?: string;          // Neighborhood/area name
}

interface PlacePhrase {
  english: string;
  local: string;
  context?: string;       // When to use this phrase
}

interface ExploreCategory {
  id: string;             // e.g., "street-food"
  emoji: string;
  label: string;
  description: string;
}
```

## API Design

### Endpoint

```
GET /destination/:code/explore/:category?page=1&limit=10
```

- `:code` — country code (e.g., "JP", "TH")
- `:category` — vibe category slug (e.g., "street-food", "hidden-gems")
- Paginated: 10 places per page
- Response: `{ places: ExplorePlace[], page: number, hasMore: boolean }`

### Gemini Prompt Strategy

One prompt per category request. The prompt includes:
- Destination country + primary city
- Category type and description
- Instruction to return real place names that existed as of training data
- Structured JSON output format
- Emphasis on local/authentic over tourist-popular
- Include disclaimer that places should be verified

### Caching

- **Server**: File-based cache with 7-day TTL (same as culture content in `server/cache/`)
- **Client**: AsyncStorage with 7-day TTL + in-memory cache (same pattern as `usePhrases.ts`)
- Cache key: `explore-${countryCode}-${category}-page${page}`

### Rate Limiting

- Same as existing cache endpoints: 60 req/min
- Gemini calls only on cache miss

## Deep Links

### "Directions" button
Opens the device's default maps app:
- **iOS**: `maps://?q=${encodedPlaceName}+${city}`  
- **Android/Web**: `https://www.google.com/maps/search/?api=1&query=${encodedPlaceName}+${city}`

Uses `Linking.openURL()` from React Native — no additional dependencies.

### "Translate" button
Navigates to the Travel tab with destination language pre-selected. Uses Expo Router's `router.push('/(tabs)/')` or updates shared context.
