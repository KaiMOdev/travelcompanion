# VoxLingo Travel Story — Design Spec

**Date:** 2026-04-05
**Status:** Draft (revised after GPT 5.4 review)
**Goal:** Reposition VoxLingo from "translation tool" to "travel companion" through phased feature sets, focused on two killer moments: **menu intelligence** and **travel survival utility** (taxi card + emergency info).

---

## Positioning

> Google Translate tells you the words. VoxLingo tells you the story.

Every feature must answer: "Does this help a traveler navigate a foreign country better?" If it's just a better translator, Google already did it for free.

**Competitive framing:** VoxLingo doesn't just compete with translation apps — it competes with the fragmented workflow travelers actually use: Google Translate + Maps + ChatGPT + TripAdvisor + screenshots + notes apps. VoxLingo consolidates the travel-language experience into one purpose-built tool.

---

## Trust & Safety Framework

AI-generated travel advice carries real risk. VoxLingo must earn trust through transparency.

### Principles

1. **Never imply certainty where none exists.** Allergen detection, cultural tips, and emergency info are AI-assisted, not guaranteed.
2. **Disclaimers at the point of action.** Not buried in settings — visible where users make decisions.
3. **Fail safe.** When confidence is low, say so. When offline, show what's cached and what isn't.

### Allergen & Dietary Disclaimers

- All allergen flags show as "Possibly contains: ..." — never definitive
- Persistent footer on menu results: "AI-detected from menu text. Does not account for unlisted ingredients or cross-contamination. Always confirm with staff if you have serious allergies."
- Confidence indicator per item: high (ingredient explicitly listed), medium (inferred from dish name/cuisine), low (uncertain)
- Items with low confidence show "Ask staff about allergens" instead of guessing

### Emergency Info Disclaimers

- Emergency numbers sourced from a static, manually verified dataset — not AI-generated
- Embassy data: show last-verified date. Link to official government travel advisory site rather than maintaining stale consulate addresses
- Persistent note: "Emergency information is for reference. Always verify with local authorities."

### Cultural Tips Quality

- Tips for hero destinations (Japan, Thailand, Italy) are editorially reviewed before launch — not raw LLM output
- All tips include a source category: "common knowledge," "local custom," or "safety advisory"
- User can flag tips as inaccurate → feeds quality review queue
- Tips avoid stereotypes by focusing on actionable etiquette, not cultural generalizations

## Phase 1 — Travel Identity + Killer Moments

**Goal:** Shift first impression from "translator" to "travel companion" AND ship the two features that justify the product's existence: smart menu camera and taxi/emergency cards.

Phase 1 is deliberately narrow — focused on the moments where VoxLingo is dramatically better than free alternatives.

### 1.1 Destination Selector

- First-open onboarding: "Where are you traveling?" — searchable list, hero destinations (Japan, Thailand, Italy) featured at top
- Selecting a destination auto-sets the target language (e.g., Japan → Japanese)
- Destination stored locally (AsyncStorage), changeable from home screen
- MVP is single destination

### 1.2 Essential Phrases

- Home screen shows a horizontally scrollable card row of 10 essential phrases for the selected destination
- Phrases for hero destinations are pre-written and editorially reviewed (not raw Gemini output)
- Non-hero destinations: Gemini-generated, cached locally, with a "Generated by AI" label
- Each phrase card shows: English phrase, translated phrase (local script + romanization), tap-to-play TTS
- Categories: greetings, ordering food, asking directions, saying thank you, emergency, polite phrases

### 1.3 Cultural Tips Card

- Below the phrase row, a "Did you know?" card surfaces 1 cultural tip at a time
- Swipe or tap to cycle through 5 tips per destination
- Tips cover: etiquette, money/tipping, food customs, safety, social norms
- Hero destination tips are editorially reviewed. Others are Gemini-generated with "AI-generated tip" label
- User can flag inaccurate tips

### 1.4 Smart Camera — Menu Intelligence (Killer Moment #1)

**This is the viral feature.** "POV: you point VoxLingo at a menu in Tokyo."

When the camera detects a menu:

- **Dish descriptions:** Each menu item gets a 1-sentence description of what it is
- **Allergen flags:** "Possibly contains: ..." with confidence indicator (see Trust Framework above)
- **Dietary tags:** Vegetarian/vegan/halal indicators where identifiable, with confidence
- **Local favorites:** Items marked as popular when Gemini can infer this
- **User dietary preferences:** Settings screen lets user set restrictions. Conflicting items dimmed with warning badge + "Ask staff" prompt

Content type detection: Gemini Vision prompt classifies input as `menu`, `sign`, or `other`. Sign mode adds a contextual explanation ("This means X — you should Y"). General mode behaves as today.

**Response format (menu):**
```json
{
  "contentType": "menu",
  "detectedLanguage": "ja",
  "items": [
    {
      "original": "カツカレー",
      "translated": "Katsu Curry",
      "description": "Breaded deep-fried pork cutlet served on Japanese curry rice",
      "possibleAllergens": ["wheat", "egg", "pork"],
      "allergenConfidence": "medium",
      "dietary": [],
      "popular": true
    }
  ],
  "disclaimer": "AI-detected from menu text. Always confirm with staff for serious allergies."
}
```

**Response format (sign):**
```json
{
  "contentType": "sign",
  "detectedLanguage": "ja",
  "originalText": "出口3 渋谷方面",
  "translatedText": "Exit 3 — Shibuya Direction",
  "context": "This exit leads toward Shibuya. Follow this sign if heading to Shibuya station."
}
```

### 1.5 Emergency Info Card (Killer Moment #2a)

- Accessible from a persistent SOS icon in the header — always one tap away
- Shows for the selected destination:
  - Police, ambulance, fire numbers (static, verified dataset)
  - "I need help" / "Please call an ambulance" / "I don't speak [language]" — pre-translated, large text, ready to show someone
  - Link to official government travel advisory for that country
- Works offline — emergency data bundled with the app for all 30 supported countries
- No embassy/consulate addresses (too hard to maintain accurately — link to official source instead)

### 1.6 Location Card for Taxi Drivers (Killer Moment #2b)

- "Show my location" button on the Travel tab
- Generates a full-screen card displaying:
  - Current address in the local language and script (reverse geocode via expo-location)
  - User's hotel/accommodation (if set in settings) in local script
  - Large, clear text — designed to hand the phone to a driver
- Requires location permission (prompted on first use of this feature, not on app open)
- Falls back gracefully: if no location permission, user can type an address and get it translated

### 1.7 App Store & Branding Update

- App subtitle: "Travel fluently, anywhere"
- Screenshots lead with menu intelligence and taxi card, not the translator
- Destination-specific screenshot sets for ASO (Japan, Thailand, Italy variants)
- Keywords: "travel translator," "menu translator," "Japan travel app," "camera translate menu"

## Phase 2 — Location Awareness & Offline

**Goal:** Make the app contextually intelligent via location and reliable without internet for critical features.

### 2.1 Country Auto-Detection

- On app open (if location permission already granted), reverse-geocode to detect country
- Auto-set target language if user hasn't manually selected one
- Show country flag + name in the header as confirmation
- If user is home (detected country matches phone locale), don't change anything

### 2.2 Offline Reliability Layer

Travelers lose connectivity at the worst moments. Critical features must work offline:

- **Emergency info:** Bundled with app (already offline in Phase 1)
- **Essential phrases:** Cached on destination selection, available offline with TTS
- **Taxi card:** Last-known location cached. Hotel address always available (stored locally)
- **Recent translations:** Last 50 translations cached locally for re-access
- **Offline indicator:** When offline, show banner: "Offline — cached phrases and emergency info available. Translation requires internet."

Storage: Use `expo-sqlite` for structured data (phrases, tips, translations, emergency info) instead of AsyncStorage. AsyncStorage for simple preferences only.

### 2.3 Translation History

- All translations saved locally (expo-sqlite)
- Organized by date, grouped by day
- Searchable by text content
- Starred/favorited phrases for quick re-access
- No export/journal features (deferred — not core value)

## Phase 3 — Monetization

**Goal:** Introduce revenue with pricing that matches episodic travel behavior.

### 3.1 Pricing Model

**Free — "Tourist":**
- Voice translation (unlimited, all 30 languages)
- Camera translation (basic — text only, no menu/sign intelligence)
- 3 cultural tips per destination
- Essential phrases (view only, no TTS on phrase cards — voice translation TTS unaffected)
- Emergency info card (free — safety should never be paywalled)

**Trip Pass — $4.99 per trip (14 days of access):**
- Smart camera (menu explanations, allergen flags, sign context)
- Unlimited cultural tips
- Phrase TTS playback + pronunciation
- Translation history + starred phrases
- Offline phrase packs
- Location card for taxis
- Activates on first use, not on purchase — no wasted trial days

**Annual Traveler — $29.99/year:**
- Everything in Trip Pass, unlimited
- For frequent travelers (2+ trips/year makes annual worthwhile)
- Multi-destination support

**Explorer — $9.99/month or $59.99/year (future):**
- Meeting mode (group translation)
- Real-time conversation mode
- Priority processing

### 3.2 Paywall Implementation

- RevenueCat for cross-platform subscription management
- Paywall shown when user taps a premium feature (smart camera result, phrase TTS)
- Trip Pass is the primary CTA — lower friction than subscription
- Trip-triggered prompts: "Heading to Tokyo? Unlock menu intelligence for $4.99"
- No free trial — Trip Pass IS the low-commitment entry point
- No ads in any tier

### 3.3 Conversion Triggers

- First camera scan of a menu → show one item with full intelligence, rest blurred → "Unlock all items for $4.99"
- Phrase card tap-to-speak → play once free, then prompt for Trip Pass
- Destination selection → "Get the full Tokyo travel kit for $4.99"

## Phase 4 — Pre-Trip & Growth (Future)

**Goal:** Extend engagement beyond the trip itself. Only build after Phase 1-3 metrics validate the core product.

### 4.1 Pre-Trip Flow

- User sets trip dates (optional)
- Push notification 3 days before trip: "Heading to [destination] soon? Practice these phrases"
- Push notification on arrival (geofence): "Welcome to [country]! VoxLingo is ready."
- Cultural briefing: longer-form, 5-minute read generated per destination

### 4.2 Growth Features

- Referral: "Send a friend a free Trip Pass" — travelers recommend to other travelers constantly
- Seasonal pricing: Discount annual plans during booking season (Jan-March)
- Creator partnerships: Affiliate codes for travel YouTubers/TikTokers

## Architecture Notes

### Engineering Complexity (Honest Assessment)

This is NOT "just prompt engineering." The full scope includes:

- Structured multimodal extraction with schema validation (smart camera)
- Content classification with confidence scoring
- Server-side caching and invalidation strategy
- Location services with reverse geocoding
- Local database (expo-sqlite) for offline data
- Subscription management (RevenueCat)
- Push notifications
- Offline asset management and sync
- Trust/safety UX patterns
- Observability and quality monitoring

Each phase should be scoped with this reality in mind.

### Gemini Integration

- **Phase 1 (phrases + tips):** New endpoints `GET /destination/:countryCode/phrases` and `/tips` — Gemini generates, server caches in memory. Hero destination content is pre-generated and reviewed.
- **Phase 1 (smart camera):** Enhanced `/vision` endpoint with content-type detection, structured response schema, confidence scoring, and disclaimer injection.
- **Phase 2 (location):** Minimal Gemini work — mostly expo-location and local data.
- **Phase 3 (monetization):** No Gemini work — client-side paywall and storage concerns.

### Caching Strategy

Gemini calls are the primary cost driver. Cache aggressively:

- Destination phrases: cache indefinitely (hero destinations are static assets, not API calls)
- Cultural tips: cache per country, refresh monthly
- Menu translations: no cache (every menu is different)
- Emergency info: static dataset bundled with app — no API calls

### Data Model

```typescript
type Destination = {
  countryCode: string;    // ISO 3166-1 alpha-2
  countryName: string;
  primaryLanguage: string; // BCP-47
  isHero: boolean;        // editorially curated content
};

type Phrase = {
  id: string;
  english: string;
  translated: string;
  romanized?: string;
  category: 'greeting' | 'food' | 'directions' | 'emergency' | 'polite' | 'shopping';
  isEditorial: boolean;   // true = human-reviewed, false = AI-generated
};

type CulturalTip = {
  id: string;
  category: 'etiquette' | 'money' | 'food' | 'safety' | 'social' | 'language';
  title: string;
  body: string;
  countryCode: string;
  sourceType: 'editorial' | 'ai-generated';
};

type MenuItem = {
  original: string;
  translated: string;
  description: string;
  possibleAllergens: string[];
  allergenConfidence: 'high' | 'medium' | 'low';
  dietary: ('vegetarian' | 'vegan' | 'halal' | 'kosher')[];
  popular: boolean;
};

type MenuTranslation = {
  contentType: 'menu';
  detectedLanguage: string;
  items: MenuItem[];
  disclaimer: string;
};

type SignTranslation = {
  contentType: 'sign';
  detectedLanguage: string;
  originalText: string;
  translatedText: string;
  context: string;
};

type EmergencyInfo = {
  countryCode: string;
  police: string;
  ambulance: string;
  fire: string;
  advisoryUrl: string;    // link to official gov travel advisory
  phrases: {
    help: string;
    callAmbulance: string;
    dontSpeak: string;
  };
};

type Translation = {
  id: string;
  originalText: string;
  translatedText: string;
  sourceLang: string;
  targetLang: string;
  timestamp: number;
  starred: boolean;
};
```

### New Dependencies

- `expo-location` — reverse geocoding for taxi card and country detection
- `expo-sqlite` — local database for phrases, tips, translations, emergency data
- `expo-notifications` — pre-trip push notifications (Phase 4)
- `react-native-purchases` (RevenueCat) — subscription/trip-pass management (Phase 3)

### Screens & Navigation

Current: 2 tabs (Travel, Camera)

After Phase 3:
- **Travel tab** — voice translator + phrase cards + tips + taxi card (home screen)
- **Camera tab** — smart camera (enhanced results for menus/signs)
- **Explore tab** (new) — destination selector, cultural briefing, emergency info

3 tabs. History is accessed from Travel tab (icon in header), not a dedicated tab.

## Go-to-Market

### Target Audience (Narrowed)

**Primary ICP:** English-speaking first-time international travelers (25-40) visiting Japan, Thailand, or Italy. They're smartphone-native, willing to pay for good apps, and active on social media. They feel anxious about the language barrier and want confidence, not just translation.

**Why these destinations:** High tourism volume, significant language barrier, rich cultural content, and strong travel creator ecosystems on TikTok/YouTube.

### Hero Message Hierarchy

- **Japan travelers:** "Decode any menu, navigate any station, survive any taxi ride"
- **Food-sensitive travelers:** "See what's really in your food — AI menu intelligence with allergen detection"
- **Solo travelers:** "Your pocket travel companion — emergency phrases, taxi cards, cultural tips"

### Launch Strategy

1. **Hero destinations at launch:** Japan, Thailand, Italy — editorially curated content, tested with real menus/signs/scenarios
2. **Pre-launch content:** TikTok/Reels showing smart camera on real menus vs Google Translate
3. **Controlled beta:** 50-100 recruited travelers heading to hero destinations. Concierge-style feedback, rapid iteration. NOT open Reddit/Discord beta (risk of negative first impressions if quality isn't ready)
4. **Public launch:** Timed for January-March (peak booking season for summer travel)
5. **Creator partnerships:** 5-10 mid-tier travel creators (50K-500K), free annual pass + affiliate code

### Key Metrics

| Metric | Target |
|--------|--------|
| Install → first translation | < 60 seconds |
| Install → destination set | > 60% in first session |
| First camera scan (menu) | > 30% of users in first trip |
| Free → Trip Pass conversion | 5-8% |
| Day 7 retention | > 30% |
| Day 30 retention | > 15% |
| Bad output flag rate | < 2% of smart camera results |
| Translations per active trip day | > 15 |
| NPS post-trip | > 50 |

- Meeting mode / multi-speaker translation (future Explorer tier)

- User accounts / social features (Firebase auth deferred)
- Language learning / spaced repetition
- Airline/hotel/eSIM partnerships
- Android Wear / Apple Watch companion
- Web app version
- Ambient location-based tips (deferred — high complexity, uncertain value, privacy concerns)
- Trip journal / PDF export (deferred — not core value)
- Embassy/consulate addresses (link to official sources instead)
